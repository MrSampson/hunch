🧠 Hunch
Changelog GitHub npm

Git stores what the code is. Hunch stores why.

A persistent, git-native reasoning graph — decisions, bugs, invariants — captured as a byproduct of commits and test failures, and surfaced to your AI coding agent over MCP. Local-first. No SaaS.

Changelog
v0.9.0
Jun 17, 2026

Time-Travel Memory + Regression Guard

  • Bi-temporal valid-time. Decisions and constraints carry a git-anchored window — query the graph --as-of any commit, tag, or branch.
  • Timeline. hunch timeline and the hunch_timeline MCP tool show what was believed and when it changed.
  • Supersede, don't delete. Replacing a decision closes its window instead of erasing it — history stays auditable.
  • Regression Guard. Flags a diff that re-introduces deliberately-retired code; fails a strict commit only when tied to a blocking invariant.
v0.8.0
Jun 16, 2026

Agent hooks + firmness

  • Grounding before every edit. A PreToolUse hook injects the file's decisions, invariants, and blast radius into context — enforced by the harness, not the model's recall.
  • Firmness, four levels. off → advisory → firm → strict; strict denies an edit that touches a blocking invariant, handing the invariant back as the reason.
  • record-constraint. A first-class write path for the invariants your code must not break.
  • Multi-assistant. hunch init wires Claude Code, Cursor, VS Code (Copilot), and Codex to one .hunch/ graph.
  • Team-native. Git merge driver resolves the graph by record id; semantic search opt-in; schema migration + compaction.
